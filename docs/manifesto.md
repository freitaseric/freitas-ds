# Freitas DS - Manifesto

O **Freitas DS** é o design system oficial criado para servir como base visual, técnica e comportamental de todos os sistemas desenvolvidos por Eric Freitas, seus futuros times de desenvolvimento e sua futura empresa de software.

Ele existe para transformar sistemas soltos em produtos consistentes, acessíveis, profissionais e reconhecíveis, sem prender cada produto a uma identidade visual fixa.

## Objetivo

O objetivo do Freitas DS é garantir que todas as interfaces desenvolvidas sob a identidade Freitas sigam os mesmos padrões de:

- aparência;
- comportamento;
- acessibilidade;
- estrutura;
- tipografia;
- espaçamento;
- navegação;
- feedback visual;
- experiência de uso.

O design system deve reduzir decisões repetidas, acelerar a construção de produtos e manter uma linguagem comum entre sistemas, times e contextos de negócio.

## Filosofia

O Freitas DS não é um tema fechado. Ele é uma base flexível para produtos diferentes compartilharem a mesma qualidade de interface.

Ele fornece:

- componentes prontos;
- tokens semânticos;
- classes utilitárias;
- padrões de layout;
- regras de uso;
- comportamento acessível;
- suporte a tema configurável.

A aplicação final define sua identidade visual por meio de seed color, tema manual, modo claro ou modo escuro. O Freitas DS define a estrutura e os papéis de design; cada sistema decide como esses papéis se expressam visualmente.

## Clareza Acima De Decoração

Toda interface deve ser fácil de entender. Elementos visuais só devem existir quando ajudam o usuário a compreender algo, tomar uma decisão ou executar uma ação.

Decoração sem função deve ser evitada. Hierarquia, contraste, espaçamento e texto claro são mais importantes do que efeitos visuais.

## Consistência Obrigatória

O mesmo problema deve ser resolvido sempre com o mesmo padrão.

Botões, campos, tabelas, menus, mensagens, estados vazios, ações destrutivas e feedbacks devem seguir regras comuns. Essa consistência reduz esforço cognitivo para usuários e reduz retrabalho para desenvolvedores.

## Conforto Visual

O Freitas DS deve permitir uso prolongado sem cansaço visual excessivo.

Para isso, deve priorizar:

- contraste adequado;
- espaçamento equilibrado;
- tipografia legível;
- estados visuais claros;
- superfícies sem excesso de ruído;
- hierarquia consistente entre conteúdo e ações.

## Acessibilidade Desde O Início

Acessibilidade não é uma etapa final. Todo componente deve nascer considerando:

- navegação por teclado;
- foco visível;
- contraste suficiente;
- estados desabilitados e inválidos;
- relação clara entre labels, mensagens de ajuda e mensagens de erro;
- comportamento previsível para tecnologias assistivas.

## Produtividade Técnica

O design system deve facilitar a vida de quem desenvolve.

Componentes devem ter APIs previsíveis, nomes claros, variações controladas e composição simples. O Freitas DS deve ajudar os times a entregar interfaces melhores com menos código repetido.

## Identidade Flexível

O Freitas DS não impõe uma cor de marca fixa.

Ele define papéis semânticos como `primary`, `surface`, `error`, `outline` e `on-primary`. Cada aplicação pode mapear esses papéis para sua própria identidade visual sem quebrar a consistência estrutural do sistema.

## Frase-Guia

**“O Freitas DS existe para construir interfaces consistentes, acessíveis e profissionais sem prender os sistemas a uma identidade visual fixa.”**
